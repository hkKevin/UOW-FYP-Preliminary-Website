Welcome to the Sydney Trains Project Site!

The Client – Sydney Trains


  • A passenger rail network serving the city of Sydney, NSW and controlled by the state government
  • Covers over 815 km and 178 stations over 8 lines
  • Runs 21 hours a day, every 3 min at major stations, every 15 minutes at most minor stations
  • 90% customer satisfaction rating with 93.4% peak punctuality and 92% average

Sydney Trains’ Issues


  • In December 2017, the NSW Government announced an overhaul of the Sydney Trains Timetable
  • Customer numbers forecast to increase by 21% over 3 years
  • Lead to increase of peak services to full track capacity
  • New timetable allows for no opportunity to recover from incidents
  • In January 2018, staff shortages and bad weather crippled the network
  • Delays spanned two days with little relief or recovery
  • This lead to an inquiry from the NSW government over the new timetable

Project Aims


  • Analyze and compare the static timetable to real-time updates and generate relevant deliverables
  • Such as tools for further analysis and visualization of decision making
  • Represent data in a creative and insightful formats
  • Provide live analysis of the current data
  • Propose a solution that will increase the punctuality of the whole network

Design Methodology - Agile


  • Allows for iterative design with fortnightly deliverables
  • Client satisfaction by continuous delivery of useful software
  • Client feedback has the ability to narrow scope and reduce miscommunications
  • Issues are raised early, leading to an early solution and a reduction in lost productivity
  • Adaption of client changes allows for higher client satisfaction
  • Scope changes can be considered properly

Development Environments


HTML, CSS & JavaScript Architecture:

  • End users of Sydney Trains will predominantly access the system through desktop
  • More rounded and uniform user experience
  • Updates and rollouts are instantaneous and do not require clients to update
  • Cross compatible between devices and operating systems
  • Streamlined development – less focus on cross-developmental issues

Database - MySQL:

  • Provides a rich query set needed for our in-depth data analysis
  • Distributive, Scalability and Redundancy features of NoSQL databases do not overcome drawbacks
  • Data collected works well in a relational model
  • ACID features highly favourable
  • Open Source

BackEnd - NodeJS:

  • Event Driven
  • Non-Blocking I/O Model
  • Lightweight & Efficient
  • Google’s V8 Engine & compilation of JavaScript directly into native machine code allows for quick processing speeds
  • Library-rich with an easy to use package manager

JSON Server:

  • Regular updates of data allows for a live-view
  • Increased page customisation from the user
  • Multi-tiered system split of interface and data retrieval
  • Allows for upgrades to be made independently
  • Versatility in data presentation and object notation

Product Promotion


  • The system allows Sydney Trains users to schedule trains more accurately
  • Preparing for potential disruptions through analysis of past data
  • Aiding Sydney Trains to ensure on time performance and reduce delays across the network
  • These benefits will be extended to customers
  • who expecting and depending on the on-time arrival and departure of their services
